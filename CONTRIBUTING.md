# Contributing

Thanks for your interest in contributing to this sample/demo repo.

## Reporting issues

- Check existing issues before opening a new one.
- Describe what happened, what you expected, and how to reproduce it.
- Include relevant screenshots, logs, or environment details when helpful.

## Submitting changes

- Fork the repository and create a branch for your change.
- Keep changes focused and easy to review.
- Test your work before opening a pull request.
- Open a pull request with a clear title and summary of the change.

## Code style

- Prefer simple, readable code over clever solutions.
- Match the style and formatting already used in the project.
- Use clear names for files, variables, and commits.
- Keep documentation concise and accurate.

This project is intended as a sample/demo repository, so contributions should
stay small, approachable, and easy for new contributors to understand.
