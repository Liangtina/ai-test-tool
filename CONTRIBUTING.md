# Contributing

Thanks for helping improve this project. Please keep contributions focused, clear, and easy to review.

## Reporting Bugs

- Check existing issues before opening a new one.
- Describe what happened, what you expected, and how to reproduce it.
- Include relevant environment details, screenshots, or logs when helpful.

## Suggesting Enhancements

- Explain the problem or use case your suggestion addresses.
- Describe the proposed change and any alternatives you considered.
- Keep requests scoped so they can be discussed and reviewed easily.

## Submitting Pull Requests

- Create a branch for your change and keep it focused on one topic.
- Update documentation or examples when behavior changes.
- Verify your changes locally before opening the pull request.
- Summarize what changed and note any testing performed.
